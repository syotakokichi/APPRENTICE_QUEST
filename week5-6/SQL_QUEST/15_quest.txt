実行計画を確認できる

1. 実行計画
SQL における実行計画とは何か、実行計画を確認すると何が良いのか、SQL 初心者にわかるように説明してください。
実行計画とは、SQLデータベースが与えられたクエリをどのように実行するかを決定するために使用する一連の手順やアルゴリズムのことです。実行計画は、データベースがクエリを最も効率的に実行する方法を見つけるために作成されます。実行計画は、様々な要素（テーブルのサイズ、インデックスの使用、結合方法など）を考慮して、データベースが最適なパフォーマンスでクエリを実行できるようにします。

実行計画を確認することが役立つ理由:
・パフォーマンスの最適化: 実行計画を確認することで、クエリが効率的に実行されているかどうかを判断できます。もし実行計画に改善の余地がある場合、インデックスの追加やクエリの書き換えなど、パフォーマンスを向上させるための方法を見つけることができます。
・問題の特定: 遅いクエリやデータベースのボトルネックを特定する際に、実行計画を調べることで問題の原因を見つける手がかりを得られます。
・クエリの理解: 実行計画を確認することで、データベースがどのようにクエリを処理しているかを理解し、それをもとにクエリの最適化やデータベース設計の改善ができます。

2. 実行計画の確認
次のクエリの実行計画を確認してください。
EXPLAIN 
SELECT * 
  FROM salaries 
 WHERE salary = 70575;

3. 実行時間
次のクエリの、最初の行・全ての行を取得するのに実際にかかった時間(ミリ秒)を確認してください。なお、EXPLAIN ANALYZE ステートメントを利用することで確認できます。
EXPLAIN ANALYZE
SELECT * 
  FROM salaries 
 WHERE salary = 70575;

4. 高速なクエリ
次のクエリの実行計画及び全ての行を取得するのに実際にかかった時間(ミリ秒)を確認してください。
EXPLAIN ANALYZE
SELECT * 
  FROM salaries 
 WHERE emp_no = 10100;

このクエリは3のクエリよりも実行時間が短いはずです。なぜこのクエリの方が高速なのかを説明してください。

このクエリの方が高速な理由は、salaryカラムにインデックス(索引)が設定されているためです。salaryカラムにインデックスが設定されているため、salaryカラムを検索する際にインデックスを使用することができます。一方、emp_noカラムにはインデックスが設定されていないため、emp_noカラムを検索する際にインデックスを使用することができません。そのため、emp_noカラムを検索する際には、全ての行を走査する必要があります。そのため、emp_noカラムを検索するクエリの方が、salaryカラムを検索するクエリよりも実行時間が長くなります。
