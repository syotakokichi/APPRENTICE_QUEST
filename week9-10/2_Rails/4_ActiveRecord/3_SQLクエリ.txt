実行される SQL クエリの確認
全データの取得をする際に実行される SQL クエリを確認してください。

Rails の ActiveRecord は SQL のクエリを意識せずに書けるのは便利ですが、一方でクエリを意識しないとパフォーマンスの悪いクエリを書いてしまうリスクもあるため、実行されるクエリを確認することは重要です。


Railsのコンソールでは、ActiveRecordメソッドを使用すると、それに対応するSQLクエリが自動的に出力されます。
例えば、Todo.allを実行した場合、次のようなSQLクエリが生成されることが確認できます。

SELECT "todos".* FROM "todos"

これは "todos" テーブルから全てのレコード（すべてのカラム）を選択するSQLクエリです。
このように、ActiveRecordを使用すると、Rubyのメソッドでデータベース操作を行うと同時に、どのようなSQLが実行されているのかも確認することができます。これにより、パフォーマンスに影響を与える可能性のあるクエリを発見し、それを最適化するためのヒントを得ることが可能です。

ただし、実際のアプリケーション開発では、SQLクエリのパフォーマンスを測定するために専用のツールを使用することが一般的です。
